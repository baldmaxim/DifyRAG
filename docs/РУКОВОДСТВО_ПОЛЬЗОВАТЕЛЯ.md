# Руководство пользователя — Document Knowledge Portal

Пошаговая инструкция: как войти, настроить, создать проект, загрузить документ,
проверить обработку и найти документ. Читается сверху вниз — от первого входа до
готового результата.

- Что это: портал управления проектами, папками, документами и версиями.
- Поиск и индексация выполняются через **Dify** (RAG), оригиналы файлов хранятся в **S3**.
- Портал сам не пишет в векторную базу — этим занимается Dify.

---

## 1. Вход и навигация

1. Откройте портал → страница **«Вход в портал»** (`/login`).
2. Введите **e-mail** и **пароль**, нажмите **«Войти»**.
3. После входа откроется **«Дашборд»**.

**Сайдбар (левое меню):**

| Пункт | Назначение |
|---|---|
| Дашборд | Общая сводка: KPI, конвейер обработки, интеграции |
| Проекты | Объекты строительства и их документы |
| Поиск | Семантический поиск по смыслу (RAG) |
| Документы | Все документы компании одним списком |
| Отделы | Подразделения и их компетенции |
| Dify Datasets | Соответствие раздел → датасет Dify (только чтение) |
| Интеграции | Состояние Dify, LM Studio, Qdrant, S3 |
| API-ключи | Ключи внешнего доступа |
| Обработка | Очередь задач индексации Dify |
| Аудит | Журнал действий |
| Настройки | Конфигурация интеграций (только admin) |

Ваша **роль** видна в меню пользователя (клик по аватару справа вверху), там же — кнопка **«Выход»**.

> **Важно про доступ.** Меню показывает все пункты всем пользователям, но действия
> ограничиваются на сервере. Если прав не хватает, страница покажет сообщение
> «нет доступа» (например, «Настройки» доступны только роли **admin**).

**Роли и что они могут** (кратко, подробнее — [PERMISSIONS_MODEL.md](PERMISSIONS_MODEL.md)):

| Роль | Возможности |
|---|---|
| admin | Всё: настройки, интеграции, API-ключи, аудит-логи, приватные датасеты + все операции с документами |
| user | Весь контент: проекты, документы (загрузка/правка/удаление), папки, отделы, реиндексация, обработка, поиск |

## 2. Начальная настройка (для admin)

Выполняется один раз перед началом работы. Если портал уже настроен — переходите к разделу 3.

1. Откройте **«Настройки»**. Слева — вкладки групп:
   **Хранилище S3, Dify, LM Studio, Qdrant, Обработка, Безопасность**.
2. На вкладке нажмите **«Редактировать»**, заполните поля, нажмите **«Сохранить»**.

**Минимум для запуска:**

| Группа | Что указать |
|---|---|
| Хранилище S3 | endpoint, регион, bucket, Access Key ID, Secret Access Key |
| Dify | «Dify включён» = вкл, адрес (Base URL), Knowledge API key |
| LM Studio | адрес, модель эмбеддингов |
| Qdrant | адрес (только для диагностики) |

3. На группах **S3 / Dify / LM Studio / Qdrant** нажмите **«Проверить»** — портал выполнит
   тест соединения и покажет статус (`ok` / `degraded` / `error`) и задержку в мс.
4. Откройте **«Интеграции»** и убедитесь, что все четыре карточки
   (Dify, LM Studio, Qdrant, S3) зелёные. Кнопка **«Проверить»** обновляет статус.

Примечания:
- Секреты вводятся один раз и не отображаются; поле помечается **«Задано / Не задано»**.
  Оставьте секрет пустым — прежнее значение не изменится.
- Тумблер **«Расширенные настройки»** открывает дополнительные параметры (таймауты, чанкинг и т.п.).
- Настройки хранятся в БД, переопределяют `.env` и применяются сразу.

Подробнее: [SETTINGS.md](SETTINGS.md), [DIFY_SETUP.md](DIFY_SETUP.md).

## 3. Создание проекта

1. Откройте **«Проекты»** → нажмите **«Новый проект»**.
2. Заполните форму:

| Поле | Обязательно | Правила |
|---|---|---|
| Код проекта | да | только `a-z`, `0-9`, `-`. Попадает в имена Dify-датасетов `project_{код}__раздел` |
| Название | да | — |
| Заказчик | нет | — |
| Адрес | нет | — |

3. Нажмите **«Создать проект»**.
4. **Дерево папок 00–99 создаётся автоматически** (`00-project-card` … `99-archive`,
   с вложенностью — например `07-finance/03-ks2-ks3/01-customer-ks2`).
5. Чтобы открыть проект — кликните по строке или иконку «глаз».

Поиск по списку проектов — по коду, названию или заказчику.

---

## 4. Загрузка документа

1. Откройте проект. Слева — **дерево папок**.
2. **Выберите папку** в дереве. Кнопка **«Загрузить документ»** активна только при выбранной папке.
3. В открывшемся окне загрузки:
   - перетащите файл в область загрузки;
   - **Название** (обязательно);
   - **Тип документа** (обязательно, из списка);
   - **Дата документа**;
   - **Контрагент**;
   - **Конфиденциальность**: `public` / `internal` / `confidential` / `restricted`
     (по умолчанию `internal`).
4. Для некоторых типов появляются дополнительные поля:
   - **КС-2 (ks2)**: период, роль стороны, сумма с НДС;
   - **УПД (upd)**: поставщик, номер документа, номер машины.
5. Нажмите **«Загрузить и отправить в обработку»**.

> **Отдел и область (scope) не заполняются вручную** — они наследуются от выбранной папки.

**Что происходит при загрузке** (портал показывает этапы):

```
Метаданные → Создан → S3 URL → Upload → Commit → В очереди
```

Файл загружается **напрямую в S3** по временной ссылке, затем ставится в очередь на индексацию
в Dify. Итоговое сообщение: «Документ загружен и поставлен в очередь на индексацию в Dify».

---

## 5. Проверка обработки (статусы Dify)

После загрузки документ проходит конвейер индексации. Статус виден в колонке **«Статус»**
таблицы документов проекта (есть фильтр: индексирован / ошибка / в очереди).

**Цепочка статусов:**

```
В очереди → Загрузка → Ожидание → Парсинг → Очистка → Разбиение → Индексация → Завершено
```

Особые статусы: **Ошибка**, **В архиве**, **Отключено**.

**Где ещё смотреть:**
- **«Обработка»** — очередь задач Dify, обновляется каждые 5 секунд. Колонки:
  Тип, Статус, Попыток, Ошибка, Обновлена. Для задач со статусом `failed` есть кнопка **«Повтор»**.
- **«Дашборд»** — плитки (проектов, документов, % проиндексировано, ошибок, задач в очереди)
  и полоса конвейера по стадиям.

Документ считается готовым, когда статус — **«Завершено»** (проиндексирован).

## 6. Действия с документом

В таблице документов проекта у каждой строки есть действия:

| Действие | Что делает | Кто может |
|---|---|---|
| **Скачать** | Открывает файл по временной ссылке (presigned URL) | все с доступом |
| **Реиндексация** | Повторно отправляет документ в Dify | manager и выше |
| **Скрыть (soft delete)** | Убирает документ из портала и из поиска Dify; **файл в S3 сохраняется** | manager и выше |
| **Заменить версию** | Повторная загрузка в тот же документ создаёт новую версию (v2, v3…) | editor и выше |

При скрытии выводится: «Документ скрыт (файл сохранён в S3)». Физически из S3 ничего не удаляется.

> **Ограничения текущего интерфейса.** Следующее работает через внешний API, но отдельного
> экрана в портале пока нет:
>
> - **История версий и откат** на предыдущую версию
>   (`GET /documents/:id/versions`, `.../versions/:versionId/make-current`);
> - **Восстановление** скрытого документа / «Корзина» (`POST /documents/:id/restore`).
>
> Как вызвать через API — см. [EXTERNAL_API.md](EXTERNAL_API.md). Пример восстановления:
>
> ```bash
> curl -X POST "$PORTAL/api/v1/documents/$DOC_ID/restore" \
>   -H "Authorization: Bearer $API_KEY"
> ```

## 7. Поиск документов

1. Откройте **«Поиск»** (семантический поиск по смыслу через Dify + Qdrant).
2. Введите запрос (не короче 2 символов), нажмите **«Найти»**.
3. Уточните фильтрами:

| Фильтр | Значения / пример |
|---|---|
| Область (scope) | project · company · people · reference · templates |
| Код проекта | код созданного проекта |
| Папка | путь, например `07-finance/03-ks2-ks3` |
| Тип документа | код типа, например `ks2` |
| Режим | **Фрагменты** (найденные куски) или **Ответ** (готовый ответ) |

4. Результаты:
   - в режиме **«Фрагменты»** — карточки с релевантностью (score): документ, проект, тип, путь, текст;
   - в режиме **«Ответ»** — сгенерированный ответ. Требует настроенного Dify App API key,
     иначе появится предупреждение.

Быстрый поиск в шапке: введите запрос и нажмите Enter — откроется «Поиск» с этим запросом.

Отдельно: **«Документы»** — общий список всех документов компании с поиском по метаданным
(название, описание, контрагент). Это обычный поиск по полям, без RAG.

## 8. Вспомогательные разделы

- **Отделы** — карточки подразделений; кнопка «Изменить» открывает редактор названия, описания
  и компетенций (skills в формате markdown).
- **Dify Datasets** — таблица соответствия «проект · раздел → датасет Dify» (только просмотр:
  датасет, область, группа, Dify ID, число документов, статус).
- **API-ключи** (admin) — **«Создать ключ»**: имя + права (scopes). **Секрет показывается один раз**
  сразу после создания — скопируйте и сохраните, повторно он не отобразится. Активный ключ
  можно **«Отозвать»**.
- **Аудит** (admin / manager) — журнал критичных действий (время, действие, ресурс, кто).
  Клик по строке — детали события (было / стало).

---

## 9. Сквозной сценарий: от начала до результата

1. **Войти** в портал.
2. *(admin, разово)* Настроить интеграции в **«Настройки»** и убедиться, что **«Интеграции»** зелёные.
3. Создать проект в **«Проекты»** (папки создадутся автоматически).
4. Открыть проект, **выбрать папку** в дереве.
5. **Загрузить документ** с метаданными.
6. Дождаться статуса **«Завершено»** (следить в таблице или на **«Обработка»**).
7. Найти документ в **«Поиск»**.
8. При ошибке — **«Повтор»** на «Обработка» или **«Реиндексация»** у документа.
